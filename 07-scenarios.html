7  Factorial Vignette Analysis Overview

7.1 Analysis Structure

Each of the seven scenario chapters that follow uses a parallel structure:

  1. Scenario Context — the vignette presented to respondents
  2. Q1: Legal Consequences — Kruskal-Wallis + Dunn’s post-hoc + VDA (figure and table)
  3. Q2: What the Law Is — Kruskal-Wallis + Dunn’s + VDA
  4. Q3: What the Law Should Be — Kruskal-Wallis + Dunn’s + VDA
  5. Δ (Delta, Q2 − Q3) — within-group Wilcoxon signed-rank tests
  6. Q4: Price estimate — Kruskal-Wallis + Dunn’s

Data: Groups A, B, C (n = 1,222) for all scenarios except Educational Use, which filters to Groups A, B1, C (n = 905) due to a supplemental data collection. See §3.1 (Codebook) for the B1 substitution.

Statistical methods are detailed in the Methods chapter. Effect sizes (VDA) are interpreted using the thresholds in Table 2.1.

Cross-reference format: figures are labeled @fig-[scenario]-q[N] (e.g., @fig-pd-q1); tables are labeled @tbl-[scenario]-q[N]-stats (e.g., @tbl-pd-q1-stats).

7.2 Analysis Pipeline

Every scenario chapter (§§7.1–7.7) calls a single function, analyze_full_scenario(), and receives back a complete set of plots, descriptive statistics, and test results. That one call sets in motion a coordinated pipeline of seven R modules, each responsible for a distinct layer of the analysis. This section documents those modules: what each one does, why it was designed the way it was, and how the pieces fit together.

All seven files live in analysis/functions/ and are sourced at book startup by _common.R. The source code for each module is folded below its description; click Show source to expand it. Code results are cached for HTML display but can be manually run as an R or Positron Notebook.

7.2.1 Pipeline Overview

The pipeline separates cleanly into three tiers: data (Module 1), statistics and visualization (Modules 2–5), and orchestration (Module 7). Module 6 supports only Chapters 4 and 5 and is never called by the scenario analysis chain.

7.2.2 Module 1: data_prep.R — Data Ingestion

data_prep.R handles everything that must happen before any statistics can be run: loading the CSV, constructing computed columns, and producing the two analysis views that the rest of the book uses.

The central design decision is single-file architecture. The two analysis views are created in _common.R by filtering on GROUP: survey_data keeps Groups A, B, and C (n = 1,222); survey_data_eu keeps Groups A, B1, and C (n = 905). The prepare_eu_data() function in this module handles the original two-file workflow and is preserved for provenance, but it is not called at book startup.

The other key contribution is delta variable creation. Each scenario includes a question about what the law currently permits (Q2) and a question about what respondents prefer (Q3). Their difference (Δ = Q2 − Q3) is stored in a column named [SCENARIO]25 (e.g., PD25). A positive delta means the law is stricter than respondents prefer; a negative delta means the law is more permissive. create_all_deltas() loops over all seven scenario codes and creates these columns in a single pass, and it gracefully skips any scenario whose Q2/Q3 columns are absent.

7.2.3 Module 2: stats_helpers.R — Statistical Calculations

stats_helpers.R provides the statistical layer: validation, descriptive summaries, hypothesis tests, and effect sizes. Its design philosophy is to keep statistical decisions in one place so that all seven scenario chapters are guaranteed to use identical methods.

The non-parametric choice is deliberate. The outcome variables are 7-point Likert scales: ordinal, bounded, and typically non-normal in distribution. Kruskal-Wallis is the appropriate omnibus test, and Dunn’s test (via rstatix::dunn_test(), Holm correction) is the appropriate post-hoc. Pairwise Wilcoxon was considered but rejected: Dunn’s test is specifically designed for post-hoc comparisons following a Kruskal-Wallis test and uses rank-sum scores from the full dataset, which gives it more appropriate statistical properties in this context.

NoteDesign note: Why Vargha & Delaney’s A over Cohen’s d or r?

Effect sizes based on means (Cohen’s d) assume interval-level data. For ordinal scales, Vargha & Delaney’s A is the recommended alternative: it is a rank-based probability that a randomly selected observation from Group X exceeds a randomly selected observation from Group Y. A value of 0.50 indicates no difference; values above 0.50 indicate Group 1 dominance, values below 0.50 indicate Group 2 dominance. Thresholds used throughout this appendix follow Vargha & Delaney (2000) and are given in Table 1.1.

The delta analysis uses a different test: test_delta_nonzero() runs a one-sample Wilcoxon signed-rank test for each group separately, testing whether that group’s median delta is significantly different from zero. A non-zero median delta indicates a gap between respondents’ perception of current law and their normative preference.

7.2.4 Module 3: plotting_themes.R — Visual Consistency

plotting_themes.R is the style guide for every figure in this Appendix. By centralizing all ggplot2 theme and argument settings, the module guarantees that every plot across all seven scenario chapters shares the same fonts, sizes, color choices, significance bracket styling, and central tendency markers.

The base theme is Tufte (via ggthemes::theme_tufte()), following Edward Tufte’s principle of maximizing the data-to-ink ratio. Unnecessary gridlines and borders are removed; the statistical signal is left to speak for itself. Font resolution is handled through the PLOT_FONT global variable, which is set in _common.R before this module is sourced; all functions default to base_family = PLOT_FONT so that the book-level font choice propagates everywhere automatically.

Two argument-list functions address the structural difference between Likert-scale questions and the price question.

get_standard_plot_args() configures the standard Q1/Q2/Q3/delta plots: no individual points shown, means displayed (parametric centrality), and significance brackets limited to statistically significant pairs.

get_price_plot_args() modifies the recipe for price data: individual estimates are shown as semi-transparent jittered dots, and the central tendency switches to median (50th percentile) because price distributions are right-skewed and means are not robust to the outliers that remain after the 99th-percentile trim.

7.2.5 Module 4: analysis_functions.R — Per-Question Plots

analysis_functions.R is where statistical tests and visualizations are actually assembled. Its four functions correspond to the four structural plot types that appear across the scenario chapters: standard questions (Q1/Q2/Q3), delta questions (Δ = Q2 − Q3), price questions (Q4), and a master dispatcher that routes each question to the appropriate specialized function.

The core plot engine throughout is ggstatsplot::ggbetweenstats(). This function was chosen because it combines group-level box plots, individual jitter (optional), pairwise significance brackets, and statistical test results in a single, opinionated call. The tradeoff is that the function’s defaults are opinionated and occasionally need to be overridden, leading to plotting_themes.R as a separate layer.

A critical detail shared by all three plot functions is group factor reordering. The raw GROUP column contains the strings "A", "B", "C", and (for EU) "B1". Left to its default, R would display these alphabetically. Each function refactors the column to display order C → A → B (Control first, then treatments in ascending information density), and maps the codes to human-readable labels: "Control (C)", "Short (A)", "Full + Primer (B)", and "Full + Primer (B1)". This ordering is applied consistently before any statistical test is run, so all group comparisons and labels in the output match this canonical ordering.

create_delta_plot() adds one step beyond the standard plot: after producing the between-group visualization, it runs a one-sample Wilcoxon signed-rank test for each group separately. The test result finding whether each group’s median delta is significantly different from zero is returned alongside the plot and appears in the scenario chapters as the Wilcoxon results table.

create_price_plot() adds outlier removal via remove_outliers_by_group() before any statistics are run. Price estimates are right-skewed and occasionally extreme; removing the top 1% within each group independently (rather than across the full sample) prevents any one group’s outliers from distorting the visual scale.

create_delta_plot() data, dv, title, test_by_group list (plot + Wilcoxon) Delta plot with per-group zero test
create_price_plot() data, dv, title, percentile list (plot + filtered data) Price plot with outlier trimming

7.2.6 Module 5: export_helpers.R — Figure Export

export_helpers.R handles the mechanics of saving analysis outputs to disk. export_plot_to_tiff() is the function called by analyze_scenario_question() whenever export_tiff = TRUE. It uses R’s base graphics device system—tiff() → print(plot) → dev.off()—to write a 300 DPI TIFF suitable for journal submission.

The default output path uses here::here("data", "figures", "v2"), which resolves relative to the project root regardless of the working directory at the time of rendering. The filename convention is [SCENARIO][QUESTION][WIDTH×10].tiff — for example, PD165.tiff for the Public Domain Q1 plot at 6.5 inches wide. The generate_filename() function encodes this convention explicitly.

The remaining functions (export_plots_batch(), save_analysis_results(), export_stats_to_csv(), export_all_results()) support the standalone batch workflow and the generate_full_report() orchestrator. They are not called during normal book rendering.

7.2.7 Module 6: randomization_checks.R — Covariate Balance

randomization_checks.R is deliberately isolated from the scenario analysis chain. Its functions appear in Chapter 4 (?sec-randomization) and Chapter 5 (Section 2.6), and they are never called by analyze_full_scenario(). Keeping balance checks in their own module prevents the scenario chapters from accumulating diagnostic overhead, and it makes the covariate-balance logic easy to find and audit independently.

check_continuous_covariate() handles numeric variables (Age, Income, Education). The test selection is adaptive: Age and Income are tested with ANOVA; scale-based variables (Internet skills) use Kruskal-Wallis. check_categorical_covariate() handles nominal variables (Race, Gender) via chi-square test. The $proportions component gives within-group proportions, which are more interpretable for balance assessment than raw counts.

The module also contains check_brand_recognition(), which analyzes the four-item brand recognition battery (Creative Commons, Byteyield, Microsoft, Mozilla). Byteyield is a fictitious brand included as an attention check; its recognition mean should be well below the 2.5 threshold.

NoteWhy isolate randomization checks from scenario analysis?

If randomization_checks.R were sourced inside each scenario chapter, every chapter would re-run the balance tests — repeating the same checks seven times with no additional information. Placing the checks in dedicated chapters (4 and 5) and in a standalone module makes the one-time nature of the diagnostic explicit.

7.2.8 Module 7: master_analysis.R — Orchestration

master_analysis.R is the top of the call stack. Its central function, analyze_full_scenario(), is what each scenario chapter calls. It takes a scenario code (e.g., "PD"), a full scenario name (e.g., "Public Domain"), the data, and an export_tiffs flag, then runs all five question types (Q1, Q2, Q3, Delta, Price) through the pipeline and collects the results into a named list.

The module’s hierarchy is explicit in the source comments: generate_full_report() → analyze_all_scenarios() → analyze_full_scenario() → analyze_scenario_question() → specialist plot functions. The Quarto book uses only the third level—analyze_full_scenario()—because each chapter handles one scenario and the book structure itself provides the “all scenarios” organization.

The EU scenario receives special handling: if scenario_code == "EU" and a data_b1 argument is provided, the function swaps in the EU-specific dataset. In the Quarto book, _common.R pre-computes both survey_data and survey_data_eu, so each scenario chapter passes the correct dataset directly.

With all seven modules loaded, every scenario chapter reduces to a single call:

The EU scenario passes survey_data_eu (Groups A/B1/C, n = 905) instead of survey_data. The pipeline’s modularity means that changing any single layer, replacing the theme in plotting_themes.R, adding an effect size in stats_helpers.R, or switching export format in export_helpers.R propagates automatically to all seven scenario chapters without touching the chapter files themselves.

7.3 How to Read the Scenario Graphs

The scenario chapters use the same ggbetweenstats engine from ggstatsplot described in §4.3, which explains the basic visual elements: the violin shape (probability density), the box plot (quartiles and median), the centrality point (large dot with numeric label), and the subtitle (omnibus test results). This section covers what is new in the scenario graphs. The key interpretive shift is directional: in Chapter 4, a non-significant result (p > 0.05) confirmed successful randomization. Here, a significant result (p < 0.05) indicates a treatment effect, statistical evidence that CC license information changed how respondents answered.

7.3.1 Group Ordering and Labels

Groups are displayed left-to-right, beginning with the control and then moving through two increasing intensities of treatment exposure to CC’s license marking: Control (C) → Short (A) → Full + Primer (B). Control respondents received a standard copyright marking or similar standard variant; Short received a brief, abbreviated CC label with logos; Full + Primer received a full text license name along with mandatory exposure to an educational primer for at least 20 seconds.

7.3.2 New Visual Elements

Two elements appear in the scenario graphs that are absent from the randomization checks:

  • Significance brackets. After the Kruskal-Wallis omnibus test, Dunn’s post-hoc test (with Holm correction) identifies which group pairs differ. Brackets connecting two groups appear above the plot, with Holm-adjusted p-values displayed at the bracket midpoint. Only statistically significant pairs are shown; if no bracket connects two groups, their difference was not significant after correction for multiple comparisons.

  • Caption with VDA effect sizes. Below each figure, the caption reports pairwise Vargha & Delaney’s A (VDA) effect sizes. VDA is a probability: it estimates the chance that a randomly selected observation from one group exceeds a randomly selected observation from the other. A value of 0.50 means no difference; departures from 0.50 indicate the direction and magnitude of the effect. Interpretation thresholds (small, medium, large) follow Vargha & Delaney (2000) and are given in Table 2.1.

7.3.3 Three Plot Types

The scenario chapters produce three structurally distinct graph types:

Q1, Q2, Q3 (Likert-scale questions). These plots display responses on a 7-point scale. Individual data points are hidden to reduce clutter. The centrality point marks the group mean, even when the test statistics are nonparametric for ease of interpretation.

Delta (Δ = Q2 − Q3). Delta plots visualize the difference between each respondent’s perception of current law (Q2) and their normative preference (Q3). A positive delta means the respondent sees the law as stricter than they prefer; a negative delta means the law is more permissive than preferred. The Y-axis is the Q2–Q3 difference rather than a raw survey response. Each delta plot is accompanied by a Wilcoxon signed-rank table reporting whether each group’s median delta is significantly different from zero — that is, whether there is a measurable gap between “what the law is” and “what the law should be” within that group.

Price (Q4). Price plots show respondents’ dollar estimates of the value of the copyrighted work in the scenario. Individual estimates are displayed as semi-transparent jittered dots because the distribution shape is substantively informative. The centrality point marks the group median rather than the mean, because price distributions are right-skewed and the median is more robust to extreme values. Before any statistics are computed, the top 1% of estimates within each group are trimmed as outliers to prevent a handful of extreme responses from distorting the visual scale and test results.

7.3.4 Quick Interpretation

For any scenario graph: if the subtitle shows p < 0.05, the groups differ overall. Check the brackets to see which specific pairs drive the difference, and consult the caption for the effect size. For delta plots, the accompanying Wilcoxon table reveals whether each group’s gap between perceived and preferred law is itself significant. The first example the reader will encounter is Figure 1 in the Public Domain scenario.

### Module 3: `plotting_themes.R` — Visual Consistency {#sec-fn-plotting-themes}

`plotting_themes.R` is the style guide for every figure in this Appendix. By centralizing all `ggplot2` theme and argument settings, the module guarantees that every plot across all
seven scenario chapters shares the same fonts, sizes, color choices, significance bracket styling, and central tendency markers.

The base theme is **Tufte** (via `ggthemes::theme_tufte()`), following Edward Tufte's principle of maximizing the data-to-ink ratio. Unnecessary gridlines and borders are removed; the statistical signal is left to speak for itself. Font resolution is handled through the `PLOT_FONT` global variable, which is set in `_common.R` before this module is sourced; all functions default to `base_family = PLOT_FONT` so that the book-level font choice propagates everywhere automatically.

Two argument-list functions address the structural difference between Likert-scale questions and the price question. 

`get_standard_plot_args()` configures the standard Q1/Q2/Q3/delta
plots: no individual points shown, means displayed (parametric centrality), and significance brackets limited to statistically significant pairs.

and the central tendency switches to **median** (50th percentile) because price distributions are right-skewed and means are not robust to the outliers that remain after the 99th-percentile

### Module 4: `analysis_functions.R` — Per-Question Plots {#sec-fn-analysis-functions}

`analysis_functions.R` is where statistical tests and visualizations are actually assembled. Its four functions correspond to the four structural plot types that appear across the
scenario chapters: standard questions (Q1/Q2/Q3), delta questions (Δ = Q2 − Q3), price questions (Q4), and a master dispatcher that routes each question to the appropriate specialized function.

The core plot engine throughout is `ggstatsplot::ggbetweenstats()`. This function was chosen because it combines group-level box plots, individual jitter (optional), pairwise significance brackets, and statistical test results in a single, opinionated call. The
tradeoff is that the function's defaults are opinionated and occasionally need to be overridden, leading to `plotting_themes.R` as a separate layer.

A critical detail shared by all three plot functions is **group factor reordering**. The raw `GROUP` column contains the strings `"A"`, `"B"`, `"C"`, and (for EU) `"B1"`. Left
to its default, R would display these alphabetically. Each function refactors the column to display order C → A → B (Control first, then treatments in ascending information density), and maps the codes to human-readable labels: `"Control (C)"`, `"Short (A)"`,
`"Full + Primer (B)"`, and `"Full + Primer (B1)"`. This ordering is applied consistently before any statistical test is run, so all group comparisons and labels in the output match this canonical ordering.

`create_delta_plot()` adds one step beyond the standard plot: after producing the between-group visualization, it runs a one-sample Wilcoxon signed-rank test for each group separately. The test result finding whether each group's median delta is significantly different from zero is returned alongside the plot and appears in the scenario chapters as the Wilcoxon results table.

`create_price_plot()` adds outlier removal via `remove_outliers_by_group()` before any statistics are run. Price estimates are right-skewed and occasionally extreme; removing
the top 1% within each group independently (rather than across the full sample) prevents any one group's outliers from distorting the visual scale.

### Module 6: `randomization_checks.R` — Covariate Balance {#sec-fn-randomization-checks}

`randomization_checks.R` is deliberately isolated from the scenario analysis chain. Its functions appear in Chapter 4 (@sec-randomization) and Chapter 5 (@sec-brand), and they
are never called by `analyze_full_scenario()`. Keeping balance checks in their own module prevents the scenario chapters from accumulating diagnostic overhead, and it makes the covariate-balance logic easy to find and audit independently.

`check_continuous_covariate()` handles numeric variables (Age, Income, Education). The test selection is adaptive: Age and Income are tested with ANOVA; scale-based variables (Internet skills) use Kruskal-Wallis. `check_categorical_covariate()` handles nominal
variables (Race, Gender) via chi-square test. The `$proportions` component gives within-group proportions, which are more interpretable for balance assessment than raw counts.

The module also contains `check_brand_recognition()`, which analyzes the four-item brand recognition battery (Creative Commons, Byteyield, Microsoft, Mozilla). Byteyield is a fictitious brand included as an attention check; its recognition mean should be well below the 2.5 threshold.

## Why isolate randomization checks from scenario analysis?

If `randomization_checks.R` were sourced inside each scenario chapter, every chapter would re-run the balance tests — repeating the same checks seven times with no additional
information. Placing the checks in dedicated chapters (4 and 5) and in a standalone module makes the one-time nature of the diagnostic explicit.

## How to Read the Scenario Graphs {#sec-howto-graphs}

The scenario chapters use the same `ggbetweenstats` engine from [`ggstatsplot`](https://indrajeetpatil.github.io/ggstatsplot/) described in [§4.3](04-randomization.qmd#sec-rand-howto), which explains the basic visual elements: the violin shape (probability density), the box plot (quartiles and median), the centrality point (large dot with numeric label), and the subtitle (omnibus test results). This section covers what is *new* in the scenario graphs. The key interpretive shift is directional: in Chapter 4, a non-significant result (*p* > 0.05) confirmed successful randomization. Here, a significant result (*p* < 0.05) indicates a treatment effect, statistical evidence that CC license information changed how respondents answered.

### Group Ordering and Labels

Groups are displayed left-to-right, beginning with the control and then moving through two increasing intensities of treatment exposure to CC's license marking: **Control (C)** → **Short (A)** → **Full + Primer (B)**. Control respondents received a standard copyright marking or similar standard variant; Short received a brief, abbreviated CC label with logos; Full + Primer received a full text license name along with mandatory exposure to an educational primer for at least 20 seconds.

### New Visual Elements

Two elements appear in the scenario graphs that are absent from the randomization checks:

- **Significance brackets.** After the Kruskal-Wallis omnibus test, Dunn's post-hoc test (with Holm correction) identifies *which* group pairs differ. Brackets connecting two groups appear above the plot, with Holm-adjusted *p*-values displayed at the bracket midpoint. Only statistically significant pairs are shown; if no bracket connects two groups, their difference was not significant after correction for multiple comparisons.

- **Caption with VDA effect sizes.** Below each figure, the caption reports pairwise Vargha & Delaney's *A* (VDA) effect sizes. VDA is a probability: it estimates the chance that a randomly selected observation from one group exceeds a randomly selected observation from the other. A value of 0.50 means no difference; departures from 0.50 indicate the direction and magnitude of the effect. Interpretation thresholds (small, medium, large) follow Vargha & Delaney (2000) and are given in [Table 2.1](01-methods.qmd#tbl-vda-thresholds).

### Three Plot Types

The scenario chapters produce three structurally distinct graph types:

**Q1, Q2, Q3 (Likert-scale questions).** These plots display responses on a 7-point scale. Individual data points are hidden to reduce clutter. The centrality point marks the group **mean**, even when the test statistics are nonparametric for ease of interpretation.

**Delta (Δ = Q2 − Q3).** Delta plots visualize the *difference* between each respondent's perception of current law (Q2) and their normative preference (Q3). A positive delta means the respondent sees the law as stricter than they prefer; a negative delta means the law is more permissive than preferred. The Y-axis is the Q2–Q3 difference rather than a raw survey response. Each delta plot is accompanied by a **Wilcoxon signed-rank table** reporting whether each group's median delta is significantly different from zero — that is, whether there is a measurable gap between "what the law is" and "what the law should be" within that group.

**Price (Q4).** Price plots show respondents' dollar estimates of the value of the copyrighted work in the scenario. Individual estimates are displayed as semi-transparent jittered dots because the distribution shape is substantively informative. The centrality point marks the group **median** rather than the mean, because price distributions are right-skewed and the median is more robust to extreme values. Before any statistics are computed, the **top 1% of estimates within each group** are trimmed as outliers to prevent a handful of extreme responses from distorting the visual scale and test results.

### Quick Interpretation

For any scenario graph: if the subtitle shows *p* < 0.05, the groups differ overall. Check the **brackets** to see which specific pairs drive the difference, and consult the **caption** for the effect size. For delta plots, the accompanying Wilcoxon table reveals whether each group's gap between perceived and preferred law is itself significant. The first example the reader will encounter is @fig-pd-q1 in the [Public Domain](_scenarios/pd.qmd) scenario.
